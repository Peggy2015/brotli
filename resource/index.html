HTML（超文本标记语言）

HTML（超文本标记语言——HyperText Markup Language）是构成Web世界的一砖一瓦。它定义了网页内容的含义和结构。除HTML之外的其他技术则通常用于描述一个网页的表现和展示效果（如CSS），或功能与行为（如JavaScript）。

“超文本”（hypertext）是指连接单个网站内或多个网站间的网页的链接。链接是网络的一个基本方面。只需将内容上传到互联网，并将其与其他人创建的页面相链接，你就成为了万维网的积极参与者。

HTML 使用“标记”（markup）来标记文本、图片和其他内容，以便于在 Web 浏览器中显示。HTML 标记包含一些特殊“元素”如<head>、<title>、<body>、<header>、<footer>、<article>、<section>、<p>、<div>、<span>、<img>、<aside>、<audio>、<canvas>、<datalist>、、、、、、、、、、<details>等等<embed>。<nav>​<output>​<progress>​<video>​<ul>​<ol>​<li>​

HTML元素通过“标签”（tag）将文本从文档中引出，标签由在“ <”和“ >”中包裹的元素名组成，HTML标签里的元素名不区分大小写。 ，小写或混合形式书写。例如，<title>标签可以写成<Title>，<TITLE>或以任何其他方式。但是，习惯上都建议将标签命名为全部小写。

高级主题

CORS处理跨域图片

通过布局crossorigin属性和适当的CORS标头，在<img>元素中定义的图片可以从外部源加载并在<canvas>元素中使用，就像从本地源加载一样。

CORS 属性

提供了一些对CORS的支持的HTML元素，比如<img>或<video>，具有crossorigin元素属性／属性（crossOrigin对象属性／属性），该属性能使你配置其跨域获取资源的请求。

使用 rel="preload" 预加载页面内容

<link>元素的rel属性的值preload允许你在 HTML 的<head>部分声明一些（资源）获取请求，以指定那些在页面加载后即需要刻刻的资源。因此在浏览器的主渲染机制介入之前，这些资源会在页面加载生命周期的早期阶段进行预加载。这样可以保证这些资源可被更早使用并减少阻塞页面首发渲染的可能性，从而提升性能。这篇文章提供了关于preload作用机制的基本指导。

HTML 属性参考

HTML元素都含有属性。这些额外的属性值可以通过各种途径对元素进行配置或调整其行为。

属性全局

全局属性可以允许在所有HTML 元素上进行设置，包括那些没有在相关标准中出现的元素。这意味着即使这些非标准元素使得文档不符合 HTML5 标准，它们也必须存在这些属性。

行级元素和块级元素

HTML元素通常是“行级”或“块级”元素。一个行级元素只会占用由定义它的标签所包围起来的空间。而一个块级元素将占用其父元素（容器）的全部空间，创建一个“块”。

链接类型

在 HTML 中，各种链接类型用于定义和定义两个文档之间的关系。可以设置链接类型的链接元素包括<a>、<area>和<link>。

网络媒体类型和格式指南

<audio>和<video>元素允许您原生地播放音频和视频媒体，无需任何外部软件的支持。
